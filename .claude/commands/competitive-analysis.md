# /competitive-analysis

冷蔵庫管理・献立提案アプリの競合を調査し、機能比較・ユーザーニーズ・ギャップ分析を行う。
結果を `docs/competitive-analysis/index.md` に保存する。過去の分析は `docs/competitive-analysis/archive/` に退避する。
**実装は行わない。** このコマンドは調査・分析のためだけに使う。

必ず 0 → 1 → 2 → 3 → 4 → 5 の順で進めること。

---

## 0. 既存ファイルの確認

`docs/competitive-analysis/index.md` が存在するか確認する。

存在する場合:

1. ファイル先頭の「最終更新」日付を読み取る
2. ユーザーに確認する:
   > 「前回の分析結果があります（最終更新: [日付]）。再調査して上書きしますか？」
3. YesならBashで既存の `index.md` を `archive/YYYY-MM-DD.md`（前回の日付）にコピーしてから次へ進む:
   ```bash
   cp docs/competitive-analysis/index.md docs/competitive-analysis/archive/YYYY-MM-DD.md
   ```

存在しない場合はそのまま次へ進む。

## 1. スコープ確認

以下を確認してから調査を開始する。

> 「調査の対象を確認します。デフォルトは以下です。変更がある場合は教えてください。
>
> - 対象市場: 国内メイン（海外アプリも参考程度に含める）
> - 調査軸: 食材管理 / 献立提案 / Claude API連携 / UX
> - 除外: レシピサイト単体（クックパッド等、在庫管理のないもの）」

---

## 2. 競合アプリのリストアップ（Agent並列実行）

以下の3つを Agent で**並列**に調査する。

### Agent A: 国内アプリ調査

WebSearch で「冷蔵庫管理アプリ 日本」「献立提案アプリ 日本 2024」等を検索し、
主要アプリ（3〜5件）の以下を収集する。

- アプリ名・提供元・プラットフォーム
- 主要機能リスト
- 料金体系（無料/有料/フリーミアム）
- ストアURL・公式サイトURL

### Agent B: 海外アプリ調査

WebSearch で「fridge management app」「meal planning app AI 2024」等を検索し、
参考になる海外アプリ（2〜3件）の同様の情報を収集する。

### Agent C: ユーザーレビュー分析

国内主要アプリのApp Store / Google Playレビューページを WebFetch で取得し、
以下を抽出・分類する。

- 繰り返し出てくる不満点（ネガティブ）
- 繰り返し出てくる要望（フィーチャーリクエスト）
- 評価されている点（ポジティブ）

3つのAgentが完了したら結果をマージして次へ進む。

---

## 3. 機能比較表の作成

収集した情報をもとに比較表を作成する。

比較軸（列）:

- 食材登録（手動/バーコード/レシート/音声）
- 在庫管理（賞味期限/残量/通知）
- 献立提案（ルールベース/AI）
- レシピ表示（手順/動画/外部連携）
- 買い物リスト連携
- 家族共有
- 料金体系

---

## 4. MealFlowとのギャップ分析

`docs/design.md` を読み、現在のMVPスコープと競合機能を照合する。

以下の3区分に整理する。

**MVPに含まれており競合と同等以上の機能**
（強みとして活かせる）

**MVPに含まれているが競合より弱い機能**
（改善優先度が高い）

**競合が持っていてMVPに含まれていない機能**

- さらに「ユーザー要望が多い（取り込む価値あり）」と「差別化不要」に分ける

## 5. docs/competitive-analysis/index.md に保存

以下のフォーマットで `docs/competitive-analysis/index.md` に書き出す。

```markdown
# 競合分析

最終更新: YYYY-MM-DD

## 調査対象アプリ

（各アプリの概要）

## 機能比較表

（マークダウンテーブル）

## ユーザーレビューから見えるニーズ

### よくある不満点

### よくある要望

### 評価されている点

## MealFlowとのギャップ分析

### 強み（MVPで優位な点）

### 改善余地（MVPで弱い点）

### 取り込む価値がある未実装機能

### 差別化不要な機能

## まとめ・示唆

（backlog検討に使えるインサイトを3〜5点）
```

保存後、まとめ・示唆をユーザーに表示して完了を伝える。
