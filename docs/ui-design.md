# UI 設計

## 方針

モバイルでの利用をメインとしつつ、デスクトップブラウザからも使えるレスポンシブ Web アプリ。
ネイティブアプリ・PWA ではなく、通常の Web アプリとして実装する。

---

## ナビゲーション

画面幅によってナビゲーションの形式を切り替える。

| 環境 | ナビゲーション | ブレークポイント |
|------|--------------|----------------|
| デスクトップ | 左サイドバー（`Sidebar`） | `sm`（640px）以上 |
| モバイル | 底部タブバー（`BottomTabBar`） | `sm` 未満 |

### タブ構成（モバイル底部）

| タブ | ラベル | リンク先 |
|------|--------|---------|
| ホーム | ホーム | `/` |
| 食品 | 食品 | `/inventory` |
| スキャン | スキャン | `/inventory/new`（バーコードモードで開く） |
| レシピ | レシピ | `/recipes` |

---

## 画面一覧

### ホーム (`/`)

**役割**: アプリのダッシュボード。期限リスクの高い食品とおすすめレシピを一目で確認できる。

**レイアウト（モバイルデザイン）**

```
[ヒーローエリア: オレンジ背景 + 挨拶 + 「今日の献立」 + + ボタン]
[期限が近い食品セクション（当日・翌日の食品リスト）]
[在庫カード（N種類の食品）]
[おすすめレシピセクション（最大3件）]
[底部タブバー]
```

**主要コンポーネント**

- `HomeExpirationSection` — 期限が近い食品リスト（最大5件、超えた場合は「もっと見る」）
- `HomeRecipeSection` — おすすめレシピ（最大3件、タイトル・理由・調理時間）

---

### 食品一覧 (`/inventory`)

**役割**: 登録済み食品の一覧表示・詳細確認・編集・削除。

**レイアウト（モバイルデザイン）**

```
[ヘッダー: 「食品一覧」 + + ボタン]
[検索バー]
[食品リスト（食品名・数量・ロット数、タップで展開）]
  └ 展開時: 消費期限 / 編集ボタン / 削除ボタン
[底部タブバー]
```

**主要コンポーネント**

- `InventoryList` — 食品サマリ一覧（期限順ソート）
- `InventoryDetailModal` — ロット詳細・編集・削除

---

### 食品登録 (`/inventory/new`)

**役割**: バーコードスキャンまたは手動入力で食品を在庫に追加する。

**レイアウト（モバイルデザイン）**

```
[ヘッダー: 「< 食品を登録」]
[タブ切り替え: バーコード | 手動入力]
[カメラエリア（バーコードモード時）]
[食品プレビューカード（スキャン結果）]
[数量・消費期限フォーム]
[「登録する」ボタン]
```

**主要コンポーネント**

- `BarcodeScanner` — カメラ映像 + リアルタイムスキャン（`@zxing/browser`）
- `FoodRegisterForm` — 食品名・数量・消費期限の入力フォーム
- `useBarcodeScanner` — スキャン状態管理 + `/api/barcode/[code]` への照合

---

### レシピ (`/recipes`)

**役割**: 在庫をもとに AI が提案したレシピ一覧の表示。

**レイアウト（モバイルデザイン）**

```
[ヘッダー: 「おすすめレシピ」 + 「在庫から N 件提案」]
[ヒーローカード（1枚目: 大きく表示、画像・食材・バッジ・「このレシピを使う」ボタン）]
[サブカード（2枚目以降: コンパクト表示）]
[底部タブバー]
```

**主要コンポーネント**

- `RecipeList` — レシピ一覧（ヒーローカード1枚 + サブカード最大2枚）
- `RecipeConfirmModal` — 食材確認 + 「このレシピを使う」アクション
