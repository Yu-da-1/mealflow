# 005 レシピ instructions の生成・管理方針を決める

**日付**: 2026-04-02
**ステータス**: Accepted

## 決定内容

seed データ実装時に Claude Code との会話の中でレシピ手順テキストを一括生成し、結果をレビューしたうえで `supabase/seed.sql` に直接埋め込む。プログラムから Claude API を呼び出すことはしない。

## 背景

既存の 35件のレシピはすべて `instructions` が NULL であり、Phase 6-4 で追加したレシピ詳細モーダルに手順が表示されない状態にある。
手順テキストをどのように用意するかを決める必要があった。

## 検討した選択肢

| 選択肢 | 概要 | 問題点 |
|--------|------|--------|
| 手書きで SQL に記述 | 35件＋α を手動で書く | 時間がかかる・品質にばらつきが出る |
| レシピ選択時に動的生成 | ユーザーがモーダルを開くたびに Claude API を呼ぶ | 毎回 API コストと待ち時間が発生する。手順は不変なので動的生成は過剰 |
| **Claude Code との会話で一括生成し seed に埋め込む** | 実装時に会話内で生成・レビューして SQL に書く。追加 API コストなし | 生成後のレビューが必要（ただし一度きり） |

## 採用理由

1. **コスト効率**: 追加の API コストがゼロ。ユーザー操作のたびに課金されることもない。
2. **表示速度**: seed に持つため、モーダル表示が DB 読み取りだけで完結する。
3. **品質管理**: 生成後に人間がレビューして SQL に確定できるため、ハルシネーションを修正できる。
4. **一貫性**: 手書きより Claude 生成のほうが文体・ステップ数の粒度が揃いやすい。

## 残課題・制約

- 生成テキストは投入前に必ず目視レビューする（明らかに間違った手順が混入しないよう）。
- 手順の粒度は「3〜5ステップ、各ステップ 1〜2 文」を目安とする。
