## 調査対象アプリ

- **DL数**: 50万以上

## ユーザーレビューから見えるニーズ

### よくある不満点

1. **食材登録が面倒・継続できない**
   - 「初期登録の量が多く、既存の食材をすべて登録するのに時間がかかる」
   - 「ズボラな人には登録がとにかく面倒」
   - 「登録したくないから買い物を抑えるという謎ループに陥った」

2. **食材データベースの限界**
   - 「アプリ上にない食材が登録できない」（pecco等で報告）
   - 「バーコード登録した商品でレシピ検索できない」

3. **レシピのバリエーション不足**
   - 「レシピが少ない・同じ提案ばかり」
   - 「手持ちの調味料では作れない料理が提案される」
   - 「レシピ機能がオマケ程度で、ネット検索と変わらない」

4. **UXの細かい問題**
   - 「数量入力がタップだと面倒、数字が小さい」
   - 「スペースを含む検索ができない」
   - 「広告が多く、使用意欲が下がる」（無料アプリ全般）

5. **技術的不安定さ**
   - 「アップデート後にデータが表示されなくなった」
   - 「データ引き継ぎがうまくいかない」

### よくある要望

1. **バーコードスキャン** - 登録の手間を減らす最大の要望
2. **Push通知** - 期限が近づいたら通知してほしい
3. **買い物リスト連携** - 献立から自動で買い物リストを生成したい
4. **家族共有** - パートナーや家族と在庫を共有したい
5. **レシピのカスタマイズ** - 嫌いな食材の除外、アレルギー対応
6. **レシート読み取り** - まとめ買い時に一括登録したい

### 評価されている点

1. **期限アラートで廃棄が減った** - 最も多いポジティブな声
2. **手持ち食材から献立を考えてくれる** - 「何を作ろう」という悩みが減る
3. **外出先から冷蔵庫の中身を確認できる** - 買い物時の重複購入防止
4. **食品ロス削減の実感** - 節約につながっているという声

## MealFlowとのギャップ分析

### 強み（MVPで優位な点）

- **期限自動設定（食品マスタベース）**: 競合は手動入力が多い中、食品マスタから期限を自動計算する仕組みはMealFlow固有の強み
- **在庫自動減算（ロット単位）**: レシピ確定時に期限が近いロットから優先的に減算する設計は競合より精度が高い
- **同レシピ連続回避ロジック**: 直近3日の提案履歴を参照して同じレシピを出さない仕組みは競合に少ない
- **Claude APIによるAI提案**: pecco等のルールベースに比べ、Claude APIを使った提案は自然言語での理由説明や柔軟性で優位

### 改善余地（MVPで弱い点）

- **食材登録のUX**: 手入力のみで登録の手間が最大の離脱原因。バーコードなどの入力補助がない分、初期設定のハードルが競合より高い
- **Push通知なし**: 期限アラートが最もユーザーに評価されている機能だが、MVPでは非対応
- **食材データベースの幅**: 食品マスタ100件は将来的に不足する可能性（競合の一部は数億品目のバーコードDBを保有）

### 取り込む価値がある未実装機能

| 機能 | ユーザー要望の多さ | 実装難易度 | 優先度 |
|------|:-:|:-:|:-:|
| Push通知（期限アラート） | ★★★ | 低〜中 | 高 |
| 買い物リスト | ★★★ | 中 | 高 |
| バーコードスキャン | ★★★ | 中〜高 | 中 |
| 家族共有 | ★★ | 高 | 中 |
| レシート読み取り | ★★ | 高 | 低 |
| 写真からの食材認識 | ★ | 非常に高 | 低 |

### 差別化不要な機能

- **動画レシピ**: YouTube・クックパッドで代替可能。開発コスト対ユーザー価値が低い
- **栄養バランス管理**: ダイエット・健康アプリと競合する領域。MealFlowのコアバリューと距離がある
- **食材の学習ライブラリ（調理用語等）**: peccoの独自要素だが、アプリの中心的価値ではない

## まとめ・示唆

1. **「登録の手間」が最大の離脱原因** — peccoをはじめ多くのアプリで「登録が面倒で続かない」が共通課題。MealFlowが勝負すべきは「初回登録のオンボーディングUX」と「日常的な更新コスト」の低減。食品マスタによる期限自動設定はこの観点で強みになる。

2. **Push通知はほぼ必須機能** — 「期限アラートで廃棄が減った」はユーザーが最も実感するポジティブな体験。MVPには入っていないが、早期に対応することで継続率が大きく変わる可能性が高い。バックログで最優先候補。

3. **買い物リスト連携が次の差別化ポイント** — 献立提案から買い物リストを自動生成する流れは、複数の競合が実装しておりユーザー要望も多い。Claude APIを活用して「この週の献立に足りない食材」を自然言語で提示できれば強い差別化になる。

4. **バーコードスキャンは登録UX改善の本命** — ユーザー要望の多さと離脱原因の分析から、バーコードスキャンは将来的に最重要機能の一つ。ただし実装難易度が高いため、まず「手入力UXの洗練」と「食品マスタの充実」で補う戦略が現実的。

5. **自然言語指示はMealFlow固有の可能性** — Ollieのみが「冷蔵庫の食材を使って」などの自然言語指示に対応。Claude APIを活用するMealFlowはこの領域でコストゼロで競合優位を作れる。「今日は疲れてる」「あっさりしたものが食べたい」といった曖昧な指示への対応を将来的に追加することで、pecco等との明確な差別化が可能。
