## やりたいこと（優先度順）

### 1. デプロイ基盤整備 ✅ backlog 移動済み（Phase 10）

CI（Phase 9）と合わせて本番環境を整える。これが完了して初めて「実際に触れるアプリ」になる。

- Vercel（Hobby）に production デプロイ
- Supabase Free プロジェクトを production 用に用意
- 環境変数の管理（local / production の分離）
- 環境: local + production の2環境のみ（staging は必要になったら追加）

### 2. 食品マスタ拡充 ✅ backlog 移動済み（Phase 11）

バーコードスキャン導入前の必須準備。オートコンプリートのヒット率を上げ、期限自動設定の精度も高まる。

- 現在100件程度 → 300件程度に増やす
- category / recipe_match_key / default_expiry_days を整備

### 3. バーコードスキャン ✅ backlog 移動済み（Phase 12）

食材登録の手間を最も減らせる機能。競合でもユーザー要望が最多。食品マスタ拡充が前提。

- カメラでバーコードを読み取り食品マスタと照合
- Open Food Facts などの外部DBとの連携も検討

### 4. Push通知（期限アラート）

競合分析で「期限アラートで廃棄が減った」が最多のポジティブ評価。継続率に直結する。

- 賞味期限が近い食品（当日・翌日）をPush通知
- Web Push または将来的にモバイル通知

### 5. 買い物リスト

ユーザー要望が多く、競合もほぼ全員実装済み。献立提案との連携で差別化できる。

- レシピ確定時に不足食材を買い物リストへ自動追加
- Claude APIで「今週の献立に足りない食材」を自然言語で提示

### 6. ユーザー認証 ✅ backlog 移動済み（Phase 13）

家族共有の前提。Supabase Authを活用し、各テーブルにuser_idを追加してRLSで分離する。

- メール/パスワード or ソーシャルログイン（Supabase Auth）
- 各テーブルに `user_id` カラム追加 + Row Level Security

### 7. 自然言語での献立指示

Claude APIを活用したMealFlow固有の差別化。海外ではOllieのみが対応。

- 「今日は疲れてる」「あっさりしたものが食べたい」などの曖昧な指示への対応
- 現在のレシピ提案UIに自然言語入力欄を追加

### 8. 家族共有

パートナー・家族と在庫・献立を共有する機能。ユーザー認証が前提。

- 複数ユーザーで同じ冷蔵庫を管理
- Supabase の Row Level Security を活用

## 検討中・保留

- **モバイルアプリ（Expo/React Native）**: design.mdに将来対応として明記。Webが安定したら検討
- **レシート読み取り（OCR）**: 実装難易度が高く、バーコードスキャンが先
- **栄養バランス管理**: コアバリューと距離があるため当面対象外
- **動画レシピ**: YouTube等で代替可能なため対象外
